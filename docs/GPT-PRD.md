# AI 생산성행동 트래킹 자동화 PRD v1.0
- Owner 팀: Product · Eng · Data
- 최종 업데이트: 2025-11-13

## 1. 개요·목표
- 문제 정의(Pain 지표 포함):
  - 일정·리소스·정산 데이터가 분절되어 동기화 비용·누락·지연 발생 → 실패 KPI: 회의 조율 시간 과다, 일정→시간→인보이스 전환 누락률, 인보이스 리드타임 증가
  - 알림·회의가 포커스 블록을 침식하고 도구 분절로 루틴 자동화 불가 → 실패 KPI: 주간 포커스 타임 비중 저하, 컨텍스트 스위칭 빈도 증가, 불필요 회의 비중 증가
  - 일정=시간기록=인보이스 파이프라인 미지원 → 실패 KPI: 미수·연체율 증가, 회수율 저하, 현금흐름 예측 오차 확대
  - 온콜·교대 로스터 분산 → 실패 KPI: 야간 알림 과다, 휴식 규칙 준수율 저하, 피로 지표 악화
- 목표(Desired Outcome 수치화):
  - 회의 조율 시간 -50% 이상, 불필요 회의 -30%
  - 주간 포커스 타임 +20–40%, 컨텍스트 스위칭 -25%
  - 인보이스 리드타임 -40%, 미수·연체율 -30–50%
  - 야간 알림 -30%, 휴식 규칙 준수율 개선
- 성공 지표(북극성/보조 KPI):
  - 북극성 KPI: 자동화 워크플로 완료율 (일정→시간기록→인보이스 무인 전환율)
    - 기준선: T0(온보딩 첫 주) 사용자별 측정값
    - 목표: 기준선 대비 +60%p (4주 이내)
    - 측정 주기: 주간
  - 보조 KPI:
    - 회의 조율 시간
      - 기준선: T0 측정, 목표: 기준선 대비 -50% 이상, 주기: 주간
    - 불필요 회의 비중
      - 기준선: T0 측정, 목표: 기준선 대비 -30%, 주기: 주간
    - 주간 포커스 타임 비중
      - 기준선: T0 측정, 목표: +20–40%p, 주기: 주간
    - 컨텍스트 스위칭 빈도
      - 기준선: T0 측정, 목표: -25%, 주기: 주간
    - 인보이스 리드타임
      - 기준선: T0 측정, 목표: -40%, 주기: 주간
    - 미수·연체율
      - 기준선: T0 측정, 목표: -30–50%, 주기: 주간
    - 야간 알림 건수
      - 기준선: T0 측정, 목표: -30%, 주기: 주간
    - 휴식 규칙 준수율
      - 기준선: T0 측정, 목표: +20%p, 주기: 주간
  - 측정 경로: 캘린더/포커스/시간기록/인보이스 로그, 알림/로스터 이벤트, 대시보드(Amplitude·BigQuery·Grafana)

## 2. 사용자와 페르소나
- G1 다시간대 협업 지식노동자: 타임존 조율·포커스 보호가 핵심 Pain
- G2 미팅 다빈도 사무직: 회의 축소·맥락 전환 최소화 필요
- G4 프리랜서/전문가: 일정=시간기록=인보이스 파이프라인, 미수/연체 리스크 감소
- G6 멀티앱 긱 워커: 수입·지출 통합과 연체 방지 알림
- 운영 리더(온콜/교대): 로스터·DND·준거성 증빙
- 페르소나 프리셋: 역할별 규칙·템플릿·리포트 기본 제공

## 3. 사용자 스토리와 수용 기준(AC)
### 3.1 타임존 자동 스케줄링
- Story: As a G1, I want meetings auto-scheduled across time zones, so that I avoid conflicts and protect focus blocks.
- AC:
  - Given 팀 캘린더와 업무 시간대가 등록됨, When 외부와 미팅 링크로 일정이 생성되면, Then p95 조율 왕복 메시지 ≤ 2회, 평균 확정 리드타임 ≤ 2시간
  - Given 포커스 블록이 존재함, When 충돌 가능한 미팅 요청이 도착하면, Then 자동으로 대체 슬롯 제안 ≥ 2개, 포커스 블록 보존율 ≥ 95%
  - Given 공휴일/근무시간 외, When 일정 생성 요청이면, Then 기본 거절 또는 다음 영업일 제안, 예외 승인 플로우 노출 (실패 케이스: 잘못된 타임존 표기 시 에러 배너 및 수정 가이드)

### 3.2 포커스 보호·회의 요약
- Story: As a G2, I want short meetings and automatic summaries, so that context switching decreases and I keep focus time high.
- AC:
  - Given 회의 길이 기본 25/50분 프리셋, When 회의 생성, Then 80% 이상 프리셋 준수, p95 실제 지속시간 ≤ 프리셋+5분
  - Given 회의 종료, When 요약 생성 트리거, Then p95 요약 생성 지연 ≤ 30초, 요약 포함 항목(의사결정/액션아이템/오너/기한) 누락률 ≤ 5%
  - Given DND(방해금지) 창구, When 알림 폭주(>10건/15분), Then 알림 묶음 처리 및 창구 종료 후 배치 노출(실패 케이스: 요약 실패 시 재시도 3회 및 수동 재생성 버튼)

### 3.3 일정=시간기록=인보이스 파이프라인
- Story: As a G4, I want calendar events to become time entries and invoices automatically, so that I reduce lead time and delinquency.
- AC:
  - Given 청구 규칙(프로젝트·요율)이 설정됨, When 일정 완료, Then 자동 시간기록 생성율 ≥ 95%, 수동 수정 UI 제공
  - Given 마감 주기(주/월), When 마감일 도달, Then 자동 인보이스 생성·발송까지 무인 처리율 ≥ 80%
  - Given 미수 발생, When 기한 초과 n일(기본 7일), Then 알림·부분 청구·재발송 플로우 제공, 회수율 ≥ 기준선+15%p (실패 케이스: 결제 실패 → 재시도·수단 변경·오프라인 안내)

## 4. 기능 요구사항(Functional, MSCW)
- Must
  - 타임존 자동 스케줄링/링크 공유, 포커스 블록·차단 규칙
  - 짧은 미팅 프리셋, 자동 요약(요약 API 연동)
  - 일정=시간기록 자동화, 마감 자동 인보이스, 기본 미수 알림
- Should
  - 현금흐름 예측·최소상환 경고, 부분 청구/수정
  - 페르소나 프리셋(규칙·템플릿·리포트), 승인 플로우·감사로그
- Could
  - 팀 온콜 로스터·우회 경로, 역할별 알림 정책, 외부 보고용 리포트
- Won’t (이번 범위 제외)
  - 급여/세무 통합, 풀 ERP, 다국가 세금 규칙 자동 완전 대응(추후 검토)
- 근거: 대안 대비 통합 파이프라인 가치(스케줄→포커스→요약→시간→인보이스)와 미수·연체 리스크 감소가 핵심 지불 의향을 형성

## 5. 비기능 요구사항(NFR) · 제약 · SLO
- 성능
  - UI p95 응답 ≤ 800ms, 백엔드 p95 API ≤ 500ms, 배치·동기화 지연 p95 ≤ 5분
  - 요약 생성 p95 ≤ 30초, 일정 생성~확정 리드타임 평균 ≤ 2시간(자율 슬롯 제안 포함)
- 신뢰성/가용성
  - 월 가용성 ≥ 99.5%(Beta), 오류율 ≤ 0.5% (5xx 기준), 데이터 손실 0
- 보안/프라이버시/준거성
  - OAuth 2.0/SCIM, 데이터 지역화(선택), PII 마스킹, 감사로그 보존 1년+
  - 온콜/휴식 규칙 증빙 이벤트 불변 로그(append-only)
- 비용
  - 요약 1건당 모델 비용 ≤ $0.01, 월 단위 사용자당 인프라 비용 한도 경보
- 모니터링·알림
  - 대시보드: 요청 지연 p95/p99, 실패율, 동기화 백로그, 요약 지연, 인보이스 파이프라인 전환율
  - 알림: 지연·오류·비용 초과 시 PagerDuty/Slack 통합
- 기술 제약
  - Google/Outlook API 쿼터, 요약 API 속도/비용, 결제 게이트웨이 가용성

## 6. 데이터·인터페이스 개요
- 핵심 엔터티
  - User, Team, CalendarEvent, FocusBlock, SummaryNote, TimeEntry, Invoice, PaymentReminder, RosterRule, AuditLog
- 주요 필드(예)
  - CalendarEvent(id, start/end, attendees, timezone, projectId)
  - TimeEntry(id, eventId, duration, rate, billable, approvedAt)
  - Invoice(id, customer, amount, dueDate, status, sentAt, paidAt)
  - SummaryNote(id, eventId, decisions[], actionItems[owner, due])
- 외부/내부 API 개요
  - Calendar: Google/Outlook 양방향 동기화(Webhook), 충돌 감지·대체 슬롯 제안
  - LLM Summary: 회의 요약 생성(입력: transcript/노트, 출력: 결정/액션/오너/기한)
  - Billing: 결제 게이트웨이(예: Stripe) 인보이스 발송/결제/재시도
  - Export: 회계용 CSV/Excel, Webhook(인보이스·결제 이벤트)

## 7. 범위(In/Out), 리스크·가정·의존성
- In Scope
  - 타임존 자동 스케줄링, 포커스 보호, 자동 요약
  - 일정=시간기록 자동화, 마감 자동 인보이스, 미수 알림
- Out of Scope
  - 급여·세무 자동 신고, 복잡한 다국가 세법 처리, 풀 온콜 라우팅 엔진
- 리스크(≥3)
  - 요약/예측 정확도 저하 → 휴먼 인 루프 검토·수정 UI, 품질 메트릭
  - 예외 처리 복잡도(부분 청구/수정) → 승인 플로우·검증 규칙
  - 초기 데이터 이관 난도 → 온보딩 마이그레이션 도구·가이드 제공
  - API 쿼터/가용성 이슈 → 캐시·리트라이·폴백 전략
- 가정
  - “연결·자동화”가 분절 도구 조합 대비 지불 의향을 만든다
- 의존성
  - Google/Outlook API, 요약 모델 API, 결제 게이트웨이, 알림/온콜 도구

## 8. 실험·롤아웃·측정
- 롤아웃
  - Beta(초기 G1/G2) → G4(프리랜서) 확장 → G6(긱) 및 팀 기능 단계 확장
- 실험 설계
  - 페르소나 프리셋 4종 A/B: 설정→첫 주→첫 인보이스/리포트 전환
  - KPI 베이스라인(T0) 측정 후 2–4주 변화율 비교: 조율 시간, 불필요 회의, 포커스 타임, 리드타임, 미수율
  - 유료 트리거(협업/인보이스/예측·알림) 가격·패키지 탄력 테스트
- 벤치마크(Competitor/Substitute 대비)
  - 조율 리드타임, 포커스 보호율, 파이프라인 자동 전환율, 미수 감소 폭 비교
  - 성공 기준(예): 파이프라인 자동 전환율 ≥ 대안 대비 +25%p, 조율 시간 -40% 이상
- 측정 도구
  - 로그/이벤트 스키마, 대시보드(Amplitude/BigQuery/Grafana), 알림(PagerDuty/Slack)
